Technical Reference

Unreal Engine 5 system architecture, recommended plugins and middleware, asset naming conventions, LOD strategy, and texture size guidelines for the Mooseville Adventures build pipeline.

⚙️

Gameplay System Flow (Technical)

Startup Sequence

  1. Load profile and save data (GameInstance + SaveGame)
  2. Stream world cells (World Partition)
  3. Start simulation systems (subsystems)
  4. Handle player actions via gameplay events
  5. Run periodic simulation ticks (crops, economy, NPC jobs)
  6. Trigger UI updates and feedback
  7. Autosave + checkpoint + continue

Recommended Architecture Pattern

GameInstance
  → Global state (save data, systems)
GameMode
  → Rules of simulation
Subsystems:
  - TimeSubsystem
  - ResourceSubsystem
  - FarmingSubsystem
  - AISubsystem
  - EconomySubsystem
Actors:
  - Buildings
  - Crops
  - NPCs
Data:
  - DataAssets / DataTables
Key principle: Use Unreal Subsystems heavily — they decouple systems cleanly. Use data-driven design early: Data Assets/Tables for crops, buildings, items, NPC jobs. Use Gameplay Tags for status and classification.
🧱

Core Systems Breakdown

🧭

1. World Exploration System

  • Handcrafted open world (not procedural)
  • Fog of war / map discovery
  • Biome system (6 biomes)
  • Character controller (3rd person)
  • AI Navigation via NavMesh
Key UE Components
  • World Partition (mandatory for 20 sq mi world)
  • Level Streaming
  • Landscape + Foliage Tool
🌱

2. Resource & Farming System

  • Resource Nodes (trees, rocks, clay pits)
  • Farming cycle: Plant → Grow → Harvest
  • Time-based or tick-based growth
  • Inventory-based yield system
Implementation Notes
  • Data Tables for: crop types, growth times, yields
  • Central TimeSubsystem drives all growth
  • Real-time or accelerated game clock
🏗️

3. Building / Placement System

  • Free placement (not grid-based)
  • Snapping where appropriate
  • Validation: collision + terrain check
Flow
Select → Preview ghost → Validate
→ Place → Deduct resources
🏙️

4. City Simulation System

  • Population / citizens (AI agents)
  • Production chains (Wheat → Flour → Bread)
  • Resource storage and distribution
  • Economy / upkeep calculations
Key Rule
Understaffed roles slow construction queues. Staffing = throughput.
🤖

5. AI System (NPCs / Workers)

  • Task-based AI (Behavior Trees + StateTree)
  • Jobs: Farmer, Builder, Gatherer, Groomer, etc.
  • Job proficiency affects output speed and quality
Flow
Assign job → Find task
→ Execute → Return result
⏰

6. Time & Simulation Manager

Critical system: Everything depends on a central time subsystem. Not centralizing time causes systems to desync.
  • Day/night cycle
  • Crop growth ticks
  • AI scheduling and routines
  • Weather and season transitions
💾

7. Save / Load System

Don't ignore this until late. Build early — retrofitting is painful.
Must Persist
  • World state (building placement, district layout)
  • Crop states (planted, growing, harvestable)
  • Player inventory and progression
  • NPC job assignments and proficiency tiers
  • Milestone status and quest progress
📦

8. Inventory System

  • Item definitions as Data Assets (preferred)
  • Stack logic for resources
  • Equipment vs. consumables separation
  • UI bindings via MVVM plugin
  • Container system for chests and storage buildings
🔌

Recommended Plugins & Middleware

🎵

Audio

Recommendation: FMOD Studio
  • Exploration ambience reacts to biome/time/weather
  • Farming and building actions: parameter-driven one-shots
  • City growth drives adaptive music intensity by district/population
  • Lighter weight than Wwise for small-to-medium teams
Also Consider
  • MetaSounds: native procedural audio, pairs well with FMOD
  • Wwise: industry standard, more powerful mixing — choose one
🌍

World & Environment

Built-In (Required)
  • World Partition — mandatory for large maps, auto-streaming
  • PCG Framework — scatter foliage, rocks, resource nodes
  • Landmass + Water plugins — terrain and water bodies
Marketplace (Highly Recommended)
  • Ultra Dynamic Sky — fast day/night + weather system
🤖

AI Systems

Built-In
  • Behavior Trees + EQS — sufficient for most NPC AI
  • StateTree — role/task logic for villagers
  • Smart Objects + ZoneGraph — structured interactions in settlements
  • Mass Entity — for many NPC workers at scale (later phase)
Add Mass/StateTree only after you have a stable core loop.
🎮

Gameplay Framework

Built-In
  • GAS (Gameplay Ability System) — tools, stamina, buffs, status effects
  • CommonUI — scalable UI architecture
  • MVVM Plugin — cleaner UI data binding
Marketplace (Optional)
  • Grid/placement building toolkit
  • Inventory framework starter
  • Advanced Locomotion System (ALS) for player movement
🎬

Animation

  • Control Rig — procedural animations (farming interactions, hoof placement)
  • ALS (Advanced Locomotion System) — good base for character movement
  • Custom bipedal moose rig required (unique character proportions)
🔧

Performance & Debug

  • DLSS / FSR / XeSS — upscaling for performance
  • Simplygon plugin — LOD generation for city assets
  • Unreal Insights — profile simulation spikes
  • Visual Logger (built-in) — AI debugging
  • Gameplay Debugger (built-in) — runtime state inspection

UV, Pivot & Collision Rules

🗺️

UV Authoring

  • Avoid tiny islands — prioritize big, clean shells
  • Minimize seams on visible faces
  • Pack UV1 (lightmap) separately from UV0 (albedo)
⭕

Pivot Placement

  • Props: logical interaction point or base center
  • Doors / gates: hinge point
  • Characters: feet at world origin (0, 0, 0)
  • Modular building pieces: corner or snap point
🔲

Collision

  • Use simple primitives — no complex per-poly collision
  • Keep gameplay collision slightly more generous than visuals
  • Hoof-friendly: no thin edge collision on ramps
⚠️

Common Pitfalls to Avoid

Architecture Pitfalls
  • ❌ Not centralizing time → systems desync
  • ❌ Hardcoding items instead of data-driven design
  • ❌ Overcomplicated AI too early (add Mass/StateTree last)
  • ❌ Ignoring save system until late in development
  • ❌ Coupling subsystems directly instead of using events
Asset Pitfalls
  • ❌ Photo textures — always hand-paint or stylize
  • ❌ High-frequency normals — keep them subtle
  • ❌ Metallic above 0.25 on non-gold assets
  • ❌ Shadow darkness below 0.3 (world goes too dark)
  • ❌ Inconsistent outline thickness between asset types
🚀

Minimum Viable Stack (Start Here)

Build in this order — don't over-engineer early:
Phase 1 — Core Foundation
  1. World Partition + Landscape
  2. Character movement (bipedal moose)
  3. Resource gathering
  4. Inventory system
  5. Time subsystem
Phase 2 — Simulation Layer
  1. Farming loop (plant, grow, harvest)
  2. Simple building placement
  3. PCG foliage and biome distribution
  4. FMOD + MetaSounds audio
  5. Save/Load system
Add AI workers, economy simulation, and NPC progression only after the core farming and building loop is stable and fun in isolation.
